  2. In part 2, can we create what is known as an "infinite" loop within the condition of the "while" loop? Describe what would or would not be necessary to achieve this. Provide a thorough explanation of your answer.

    3. Yes, it’ possible to create an infinite loop within the condition of a while loop. An infinite loop means that the loop never terminates on its own, and in JavaScript this happens when the condition is always true. We could create a while loop where the condition is true and then have an if loop inside of it that would guess the number. Here the condition will never become false, so the loop runs until we use aa break statement to stop it. This is a safe approach as we still have control by breaking out when the correct condition is met. Therefore, to achieve this infinite loop the condition must always be evaluated to true, for example: while(true). You must also have a way to exit the loop for it to be successful, for example with a break statement inside the loop.

  4. Explain the relationship between rows and columns in nested "for" loops. Can we, for instance, change the order, considering rows to be innermost or columns to be innermost, or does this not matter?

    5. In nested for loops, we have an outer loop and inner loop. Tehe outer loop controls what usually is the rows, and the inner loop controls the columns. For example, in part 5 the outer loop is for the rows, and the inner loop is for the columns, this is also where the text for each row is created. Changing the order of the loops will change how the data or output is generated. If columns are the outer loop and rows are the inner loop, the output would build column by column instead of row by row. Which may produce a different visual or logical result, depending on the application. Therefore, the order matters if the relationship between the rows and the columns is significant, such as printing out a table. For tasks where the order does not affect the result, the loops could theoretically be swapped without it having too much meaning. But unless there is a reason to swap them around, I would stick to the “normal” approach which would be the outer loop for rows and the inner loop for the columns.
