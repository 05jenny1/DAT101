  1. this assignment, we use the DOM. Explain how this library can help us develop interactive web pages.

    2. The Document Object Model (DOM) is the structure that the browser creates from an HTML page. It is a built-in interface that allows JavaScript to access and change the webpage while it is running. The DOM represents all the elements as objects that JavaScript can work with.

    By using the DOM, we can read values from input fields, update text, change styles, create new elements, or remove elements. The DOM is what makes interactive web pages possible, because it lets JavaScript communicate directly with HTML.

  2. What is an event? Simply explain what happens when we add an event listener to an object.

    3. An event is something that happens in the browser, for example a mouse click, a key press, moving the mouse, or selecting something in a dropdown.

    When we add an event listener to an object, we tell the browser to “listen” for a specific event on that element. When the event occurs, the browser automatically runs the function we have connected to that event.

  3. Sometimes we need to use an event object to get data from the event. Explain how this event object is structured, and how to retrieve data from a keyboard or mouse.

    4. When an event occurs, the browser creates an event object. This object contains information about what happened.

    For keyboard events, the event object includes which key was pressed, for example event.key. For mouse events, the event object includes the mouse position on the screen.

    This event object is automatically sent to the event listener’s function, so we can read the information inside it and use it in our program.

  4. Explain the main difference between DOM id and DOM name. Can we have identical IDs?

    5. The main difference between id and name in the DOM is that an id must be unique. It identifies one specific element on the page, and JavaScript functions like getElementById rely on this uniqueness.

    A name, however, can be shared by multiple elements. This is useful for grouping elements, such as radio buttons that belong to the same group, like in part 4 of the assignment.

    Because of this, we cannot have identical IDs in the same HTML document. If two elements have the same ID, the browser would not know which one to return, and it breaks the DOM rules. Names, however, can and often should be shared.
